1 الفكرة الأساسية

Git بيعتمد على 3 عناصر أساسية:

  • user.name → اسمك في الـ commit
  • user.email → إيميلك في الـ commit
  • SSH Key → هويتك الحقيقية عند GitHub
Git Configمين كتب الكود
←
SSH Keyمين مسموح له يرفع
←
Remote URLأي حساب يُستخدم
💡 تبسيط سينيور:
Git = "مين كتب الكود"  |  GitHub = "مين مسموح له يرفع"

2 إعداد الحساب الأساسي (Global)

git config --global user.name "Eslam Work"
git config --global user.email "work@email.com"

📌 شرح كل جزء

  • git config → تعديل إعدادات Git
  • --global → على كل المشاريع
  • user.name → اسمك في الـ commits
  • user.email → الإيميل في الـ commits
💡 ده الحساب الافتراضي — غالبًا حساب الشغل

3 Override لحساب شخصي (Local)

git config user.name "Eslam Personal"
git config user.email "personal@email.com"

📌 الفرق

  • بدون --global → يبقى خاص بالمشروع الحالي فقط
  • Local بيتغلب على Global داخل نفس الـ repo
💡 أي repo شخصي هياخد البيانات دي بدل الـ global

4 إعداد SSH لكل حساب

1️⃣ إنشاء مفتاح (مُفضّل: Ed25519)

# حساب الشغل
ssh-keygen -t ed25519 -C "work@email.com" -f ~/.ssh/id_ed25519_work

# الحساب الشخصي
ssh-keygen -t ed25519 -C "personal@email.com" -f ~/.ssh/id_ed25519_personal
  • -t ed25519 → أسرع وأأمن من RSA (مُوصى به)
  • -f → تحديد اسم ومسار الملف
  • -C → تعليق (عادة الإيميل) للتمييز
📌 بديل RSA: ssh-keygen -t rsa -b 4096 -C "email@example.com"

2️⃣ عرض المفتاح وإضافته لـ GitHub

cat ~/.ssh/id_ed25519_work.pub
# أو
pbcopy < ~/.ssh/id_ed25519_work.pub   # macOS — ينسخ مباشرة
💡 انسخ المفتاح → GitHub → Settings → SSH and GPG keys → New SSH key

3️⃣ اختبار الاتصال

ssh -T git@github.com
# المفروض يرد: Hi username! You've successfully authenticated...

5 إعداد SSH Config (أهم خطوة)

nano ~/.ssh/config
# Work (default)
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_work
    IdentitiesOnly yes

# Personal
Host github-personal
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_personal
    IdentitiesOnly yes

📌 شرح

  • Host → alias نستخدمه في الـ remote URL
  • IdentityFile → المفتاح المستخدم لهذا الـ host
  • IdentitiesOnly yes → يمنع SSH من تجربة مفاتيح تانية
💡 github.com → حساب الشغل  |  github-personal → الحساب الشخصي
⚠️ تأكد إن صلاحيات الملفات صح:
chmod 600 ~/.ssh/config و chmod 600 ~/.ssh/id_*

6 ربط كل Repo بالحساب الصح

✅ مشروع شغل

git remote set-url origin git@github.com:company/repo.git

✅ مشروع شخصي

git remote set-url origin git@github-personal:username/repo.git
💡 الـ Remote URL هو اللي بيحدد أي SSH key يُستخدم — مش الـ git config بس

🔍 التحقق من الـ remote الحالي

git remote -v

7 رفع المشروع

git add .
git commit -m "your message"
git push -u origin main

📌 شرح

  • add → تجهيز الملفات للـ commit
  • commit → حفظ نسخة محلية
  • push -u origin main → رفع + ربط الفرع بالـ remote
📌 لو الفرع اسمه master مش main، غيّر الأمر accordingly

8 أوامر الفحص السريع

# حالة المشروع
git status

# الفروع
git branch

# الـ remote URLs
git remote -v

# إعدادات الحساب في المشروع الحالي
git config user.name
git config user.email

# كل الإعدادات (global + local)
git config --list --show-origin
💡 --show-origin بيقولك كل setting جاي من فين — global ولا local

9 أشهر المشاكل وحلولها

1️⃣ refspec error

error: src refspec main does not match any
💡 السبب: مفيش commit بعد — اعمل git commit الأول

2️⃣ Permission denied (حساب غلط)

ERROR: Permission to user/repo.git denied to other-user
💡 الحل: SSH بيستخدم key غلط — راجع ~/.ssh/config والـ remote URL

3️⃣ Commit بإيميل غلط

💡 الحل: اعمل local config في الـ repo:
git config user.email "correct@email.com"

4️⃣ Host key verification failed

💡 الحل: ssh-keyscan github.com >> ~/.ssh/known_hosts

10 إعداد متقدم: Conditional Includes

بدل ما تعمل local config في كل repo شخصي، ممكن تAutomate بالمجلد:

# في ~/.gitconfig
[user]
    name = Eslam Work
    email = work@email.com

[includeIf "gitdir:~/personal/"]
    path = ~/.gitconfig-personal
# في ~/.gitconfig-personal
[user]
    name = Eslam Personal
    email = personal@email.com
💡 أي repo داخل ~/personal/ هياخد الإعدادات الشخصية تلقائيًا

11 Checklist قبل الـ Push

  • ✓ عملت commit (git status نظيف)
  • ✓ user.name و user.email صح في المشروع
  • ✓ الـ remote URL بيستخدم الـ host الصح (github.com vs github-personal)
  • ✓ SSH key مضاف في GitHub للحساب المناسب
  • ✓ ssh -T بيرد باسم الحساب المتوقع
